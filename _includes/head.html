{%- seo -%} {%- comment -%} site.css, not style.css. The github-pages gem defaults `theme` to jekyll-theme-primer even with no `theme:` key in _config.yml, and that theme ships assets/css/style.scss -- which compiles to the same output path a file named assets/css/style.css would occupy. Sharing the path meant a full build wrote ours last, but every --watch regeneration re-rendered the theme's SCSS over it without re-copying our static file, so the CSS vanished on the first refresh after any edit. Owning a filename the theme does not use avoids the collision entirely. {%- endcomment -%} {%- feed_meta -%}
